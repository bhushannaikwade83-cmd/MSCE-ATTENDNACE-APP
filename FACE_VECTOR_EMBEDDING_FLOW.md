  - Function: `register_face()`

1. User takes photo of student's face

   - `student_id`

   - `name`

## 🎯 Step 2: Marking Attendance (Face Matching)

### Flow:
```
Mark Attendance Screen
    ↓
Select Student (Roll Number)
    ↓
Take Face Photo (Camera)
    ↓
ArcFaceBackendService.verifyStudentFace()
    ↓
Backend API: /api/v1/verify
    ↓
Generate 512-dim Vector Embedding
    ↓
Match with Stored Vector (1:1 comparison)
    ↓
Calculate Similarity Score
    ↓
✅ Match if similarity ≥ 70% (threshold)
```

### Code Location:
- **Flutter**: `lib/presentation/screens/admin_attendance_screen.dart`
  - Function: `_markAttendance()` → `ArcFaceBackendService.verifyStudentFace()`
  
- **Backend**: `backend_api/main.py`
  - Endpoint: `POST /api/v1/verify`
  - Function: `verify_face()`
  
- **Vector Matching**: `backend_api/vector_db.py`
  - Function: `get_vector_by_roll()` - Gets stored vector for roll number
  - Function: `calculate_similarity()` - Compares embeddings

### What Happens:
1. User selects student roll number
2. User takes photo of student's face
3. Photo sent to backend as base64
4. Backend generates **512-dimensional vector embedding**
5. Backend retrieves **stored vector** for that roll number
6. Backend calculates **similarity** (cosine similarity)
7. If similarity ≥ 70% → ✅ **Match** → Attendance marked
8. If similarity < 70% → ❌ **No match** → Attendance blocked

## 📊 Vector Embedding Details

### What is a Vector Embedding?
- **512-dimensional array** of numbers (e.g., `[0.123, -0.456, 0.789, ...]`)
- Represents facial features in mathematical form
- Generated by **DeepFace** AI model
- Each student has a **unique vector** (like a fingerprint)

### Example:
```python
# Student 1 (Roll: 101)
embedding_1 = [0.123, -0.456, 0.789, ..., 0.234]  # 512 numbers

# Student 2 (Roll: 102)
embedding_2 = [0.456, -0.123, 0.567, ..., 0.789]  # Different 512 numbers
```

### Storage:
- Stored in **FAISS vector database** (fast similarity search)
- Each vector linked to student metadata:
  ```json
  {
    "student_id": "abc123",
    "roll_number": "101",
    "name": "John Doe",
    "institute_id": "inst_001"
  }
  ```

## 🎯 Matching Process

### Similarity Calculation:
```python
similarity = cosine_similarity(attendance_embedding, stored_embedding)
# Returns value between 0.0 and 1.0
# 1.0 = Perfect match
# 0.0 = No match
```

### Thresholds:
- **Registration**: No threshold (just stores the vector)
- **Verification (1:1)**: **70%** similarity required
- **Recognition (1:N)**: **85%** similarity required

### Example:
```
Attendance Photo Embedding: [0.120, -0.450, 0.780, ...]
Stored Student Embedding:    [0.123, -0.456, 0.789, ...]
Similarity: 0.92 (92%)

Result: ✅ MATCH (92% > 70% threshold)
```
