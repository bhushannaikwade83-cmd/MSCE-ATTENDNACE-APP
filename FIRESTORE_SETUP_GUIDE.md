# Firestore Setup Guide for New Features

This guide explains what needs to be added/configured in Firestore for all the new features.

## 📋 New Collections Required

### 1. **Subjects Collection**
**Path:** `institutes/{instituteId}/subjects/{subjectId}`

**Structure:**
```json
{
  "name": "GCC-TBC ENGLISH 30 WPM",
  "code": "GCC-TBC_ENGLISH_30_WPM",
  "createdAt": "2026-01-15T10:30:00Z",
  "isActive": true,
  "updatedAt": "2026-01-15T10:30:00Z" (optional)
}
```

**Fields:**
- `name` (string): Subject name (e.g., "GCC-TBC ENGLISH 30 WPM")
- `code` (string): Auto-generated code (e.g., "GCC-TBC_ENGLISH_30_WPM")
- `createdAt` (timestamp): Creation timestamp
- `isActive` (boolean): Whether subject is active (true/false)
- `updatedAt` (timestamp, optional): Last update timestamp
- `deletedAt` (timestamp, optional): Soft delete timestamp

**Note:** This collection is auto-created when you first use the app. Default subjects are automatically initialized.

**Note:**

- `batchIds` is an array for multiple batches
- `subject` (singular) is kept for backward compatibility
- `subjects` (plural) is an array for multiple subjects

    }

  // Other fields...
  "batchId": "batch123",
  "batchName": "Batch 1",
  "batchTiming": "08:00 - 09:00",

- `"pending"`: Entry photo taken, waiting for exit photo
- `"present"`: Both entry and exit photos taken
- `"absent"`: No entry photo or left without exit photo

## 🔒 Firestore Security Rules

### Required Rules for New Collections

Add these rules to your `firestore.rules`:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Institutes collection
    match /institutes/{instituteId} {
      
      // Subjects subcollection - NEW
      match /subjects/{subjectId} {
        // Allow read for authenticated users
        allow read: if request.auth != null;
        // Allow write only for admins
        allow write: if request.auth != null 
          && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      }
      
      // Batches subcollection (updated)
      match /batches/{batchId} {
        allow read: if request.auth != null;
        allow write: if request.auth != null 
          && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      }
      
      // Students subcollection (updated)
      match /students/{studentId} {
        allow read: if request.auth != null;
        allow write: if request.auth != null 
          && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      }
      
      // Attendance subcollection (updated)
      match /attendance/{attendanceId} {
        allow read: if request.auth != null;
        allow write: if request.auth != null 
          && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      }
    }
  }
}
```

## 📊 Firestore Indexes

### Required Composite Indexes

You may need to create these indexes in Firebase Console:

1. **Attendance by Date Range:**
   - Collection: `institutes/{instituteId}/attendance`
   - Fields: `date` (Ascending), `status` (Ascending)
   - Query scope: Collection

2. **Attendance by Subject:**
   - Collection: `institutes/{instituteId}/attendance`
   - Fields: `date` (Ascending), `subjects` (Array Contains)
   - Query scope: Collection

3. **Batches by Semester and Year:**
   - Collection: `institutes/{instituteId}/batches`
   - Fields: `year` (Ascending), `semester` (Ascending)
   - Query scope: Collection

4. **Students by Semester:**
   - Collection: `institutes/{instituteId}/students`
   - Fields: `semester` (Ascending), `createdAt` (Descending)
   - Query scope: Collection

## 🔍 Verification

To verify everything is set up correctly:

1. **Check Subjects:**
   - Go to Batch Management → Auto-Generate
   - Subjects should load automatically
   - If empty, they will be initialized on first use

2. **Check Batches:**
   - Use Auto-Generate feature
   - Check Firestore Console → `institutes/{instituteId}/batches`
   - Should see batches with `semester` and `isAutoGenerated` fields

3. **Check Students:**
   - Add a new student
   - Check Firestore Console → `institutes/{instituteId}/students`
   - Should see `semester`, `batchIds`, and `subjects` fields

4. **Check Attendance:**
   - Mark attendance (entry and exit)
   - Check Firestore Console → `institutes/{instituteId}/attendance`
   - Should see `entryPhoto`, `entryTime`, `exitPhoto`, `exitTime` fields

## 🆘 Troubleshooting

**If subjects don't load:**
- Check Firestore security rules allow read access
- Check if subjects collection exists
- Try adding a subject manually in Firestore Console

**If auto-generate doesn't work:**
- Check Firestore security rules allow write access
- Check if batches collection exists
- Verify institute ID is correct

**If attendance photos don't show:**
- Check B2 storage permissions
- Verify photo URLs are stored correctly
- Check `entryPhotoPath` and `exitPhotoPath` fields exist
