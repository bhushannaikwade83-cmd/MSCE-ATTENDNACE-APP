TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════════════════════

Brevo Transactional API Endpoint:
  POST https://api.brevo.com/v3/smtp/email

  Request Headers:
  - accept: application/json
  - api-key: [BREVO_API_KEY]
  - content-type: application/json

  Request Body:
  {
    "sender": {
      "name": "EDUSETU",
      "email": "noreply@edusetu.com"
    },
    "to": [
      {"email": "[admin_email]"}
    ],
    "subject": "Your OTP for EDUSETU",
    "htmlContent": "<h2>Your OTP: <strong>[OTP]</strong></h2>..."
  }

  Response:
  - Status 201: Email queued successfully
  - Status 400-500: Error (check Brevo logs)

Fallback Edge Function:
  Function: email-otp
  Endpoint: Supabase Edge Functions
  Params: mode, to, otp, purpose
  Rate: Limited per deployment
  Note: Use if Brevo transactional fails
